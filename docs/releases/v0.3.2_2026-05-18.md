# v0.3.2 - 2026-05-18

First LLM Studio v0.3.2 tightens the fine-tune experiment loop, makes local verification faster to run in partitions, and fixes screenshot smoke so release evidence captures the actual web UI instead of the desktop.

## Highlights

- Added runnable fine-tune operation closures for evaluation, adapter chat, adapter export, and distillation starter data.
- Added an operation history panel so completed fine-tune actions keep metrics, artifacts, and errors in one audit trail.
- Expanded the LLaMA-Factory parity roadmap into concrete Train, Evaluate & Predict, Chat Adapter, Export, and Distillation slices.
- Split TypeScript verification into smaller core and Agent partitions with progress heartbeats and profiling mode.
- Fixed screenshot smoke to use Node 22 plus Playwright against system Chrome, producing real `/agent` and `/admin` screenshots.
- Refreshed public screenshots for GitHub and launch documentation.

## Fine-tune

- `POST /api/admin/finetune` now supports:
  - `run-evaluation`
  - `run-chat-adapter`
  - `run-export-adapter`
  - `run-distillation`
- Fine-tune operations persist as first-class records with metrics and artifact paths.
- The UI exposes direct action buttons in Evaluate & Predict, Chat Adapter, and Export panels.
- Distillation output can create a starter dataset record for follow-up training.

## Verification Infrastructure

- Added fine-grained typecheck projects for core shared modules, i18n, demo data, Agent core, fine-tune core, community discovery, scripts, Agent API, and Agent UI.
- `npm run typecheck:changed` now maps changed files to the smallest relevant partition set.
- `npm run smoke:screenshots` no longer silently falls back to invalid desktop captures when Playwright is available.
