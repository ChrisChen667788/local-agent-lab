# v0.2.x Cadence

## Goal

Keep `v0.2.x` on a small-step release rhythm:

- one stable node at a time
- no large structural rewrites
- benchmark and runtime changes always leave a short verification note

## Default rhythm

1. Collect a small batch of related fixes
2. Run:
   - `./node_modules/.bin/tsc --noEmit`
   - `./scripts/smoke-test.sh`
3. If runtime or benchmark changed, record one benchmark comparison
4. Write or update the next release note draft
5. Cut a release only when the worktree is clean and the app is reachable on:
   - `http://localhost:3011/agent`
   - `http://localhost:3011/admin`

## What belongs in v0.2.x

- UI density and readability polish
- benchmark execution stability
- runtime observability
- export consistency
- knowledge-base maintenance UX
- release discipline improvements

## What should wait for v0.3.0

- major server-side persistence changes
- second-stage retrieval architecture
- broader agent orchestration redesign
- schema-breaking export changes
