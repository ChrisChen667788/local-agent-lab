# v0.3.0 Preread

## Current baseline

`v0.2.x` has already stabilized:

- `/agent` and `/admin` as the main product surfaces
- local/remote benchmark separation
- benchmark progress recovery and failure diagnostics
- local runtime phase visibility
- knowledge-base CRUD plus path import
- release notes, smoke checks, and stable-node versioning

## Candidate v0.3.0 themes

### 1. Server-side persistence

- durable session storage beyond browser-local state
- server-side export bundles
- resumable multi-device state where it makes sense

### 2. Retrieval stage two

- stronger import pipeline
- richer chunk navigation and source traceability
- optional hybrid retrieval and rerank
- clearer separation between grounded answers and general answers

### 3. Agent workflow uplift

- more explicit planner execution state
- better tool orchestration summaries
- stronger failure recovery across long-running tasks

## Gate before starting v0.3.0

- `v0.2.x` smoke path stays green
- benchmark remote full-suite remains stable across repeated runs
- local benchmark baselines are refreshed
- release cadence stays small and reversible

## Open question

The main decision for `v0.3.0` is whether we want the headline to be:

1. persistence-first
2. retrieval-first
3. agent-orchestration-first

Until that decision is clear, `v0.2.x` should continue to absorb only incremental, low-risk polish.
