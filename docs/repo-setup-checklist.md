# GitHub Repo Setup Checklist

Use this checklist when setting up a new repository for social preview, pinned repo display, and contributor onboarding.

## Social Preview

- [ ] Add a social preview image (1280×640 px recommended) at **Settings → General → Social preview**
- [ ] Verify the preview renders correctly when sharing the repo link on social platforms
- [ ] Include the project name and a brief tagline on the image

## Pinned Repos

- [ ] Pin up to 6 repos on your GitHub profile that best represent your work
- [ ] Ensure each pinned repo has:
  - [ ] A clear, descriptive `description` field
  - [ ] A topic/tag list (at least 3 topics)
  - [ ] An up-to-date `README.md` with badges and screenshots

## README Essentials

- [ ] Project title and one-line description
- [ ] Badges (build status, license, version, platform)
- [ ] Installation instructions
- [ ] Usage examples
- [ ] Contributing guidelines (link to `CONTRIBUTING.md`)
- [ ] License file (`LICENSE`)
- [ ] Screenshots or demo GIF if applicable

## Community Files

- [ ] `CONTRIBUTING.md` — how to contribute
- [ ] `CODE_OF_CONDUCT.md` — community standards
- [ ] `SECURITY.md` — vulnerability reporting
- [ ] Issue templates (bug report, feature request)
- [ ] Pull request template

## GitHub Settings

- [ ] Enable Discussions (if community engagement is desired)
- [ ] Enable GitHub Actions or CI
- [ ] Set default branch to `main`
- [ ] Configure branch protection rules for the default branch
- [ ] Add repository topics under **About → Topics**

## Release & Distribution

- [ ] Create initial release tag (e.g., `v0.1.0`)
- [ ] Add release notes with changelog
- [ ] Attach binaries or distribution artifacts if applicable
